# Touchpad pinch-to-zoom on Wayland — research context

Investigation into delivering real touchpad pinch gestures (libinput
`pointer-gesture-pinch`) to godbml under Hyprland. Captured here so the
decision tree doesn't have to be re-walked.

## Problem

Switching to Hyprland surfaced two things:

1. Ebiten on Linux runs through XWayland (no native Wayland backend) and
   spams `XGB: conn.go: Could not get authority info: EOF` warnings.
2. Native touchpad pinch-to-zoom does not work — Ebiten exposes no gesture
   API and X11 has no wire protocol for gestures, so XWayland drops them.

The XGB warning is cosmetic. The pinch gap is the load-bearing one.

## Why no Go toolkit currently delivers this

The architectural rule on Linux: **apps don't read `/dev/input`. The
compositor does, and republishes synthesized gestures over the
`wp_pointer_gestures_v1` Wayland protocol.** GTK4, Qt6, Chromium all subscribe
to it. None of the Go toolkits do.

| Toolkit | Wayland window | Subscribes to `wp_pointer_gestures_v1`? |
|---|---|---|
| **Ebiten** v2.9.9 | No — vendored GLFW is X11-only. Native Wayland was deleted in v2.6.0 (issue #2759) and won't return until the GLFW port to pure Go lands (#2284, no progress). | No |
| **Gio** v0.9.0 | Yes (working backend in `app/os_wayland.{c,go}`). | No — `grep -rln "pointer_gestures\|pinch\|swipe"` in `app/` returns zero matches. |
| **Fyne** | Yes (build tag) | No |
| **Raylib-go** | Yes (with `-tags wayland`, via system GLFW's experimental Wayland backend) | No — but raylib has its own gesture synthesis layer (`rgestures.h`) that would fire IF GLFW delivered touch events. GLFW does deliver `wl_touch` (touchscreens) but not `wp_pointer_gestures_v1` (touchpads) — open issue glfw/glfw#90 since 2013, PR #2419 since 2023, unmerged. |

So no upstream gives this for free. Either we accept Ctrl+Wheel zoom as the
universal convention, or we add the protocol subscription ourselves.

## Why "just bolt evdev on" was rejected

Reading `/dev/input/event*` directly with a pure-Go evdev library works on X11
and Wayland, but `/dev/input/event*` is `root:input 0660`. Shipping a desktop
app that requires `usermod -aG input` or `sudo` is broken UX. Path discarded.
Probe lives at `cmd/pinch-probe/` for reference.

## Why Gio was rejected as a destination

Built a probe at `cmd/gio-probe/` (now deleted) that opened a native Wayland
Gio window. Two issues:

1. Confirmed empirically that Gio surfaces no pinch events. `pointer.Source`
   has only `Mouse` / `Touch` (no `Touchpad`); `pointer.Kind` has no `Pinch`;
   the Wayland backend doesn't bind `wp_pointer_gestures_v1`.
2. **Bigger:** Gio is event-driven retained-mode. By default it only redraws
   on event arrival — no continuous tick. Pan/zoom feel jerky compared to
   Ebiten's fixed-tick loop. Canvas-style apps want a game-engine paradigm
   (Figma/Tldraw/Excalidraw all run on game-loop renderers or
   `requestAnimationFrame`).

## The technique that does work

`wp_pointer_gestures_v1` events fire when the **client** has surface focus,
not when a specific `wl_pointer` does. So:

- Use a toolkit that has a Wayland backend (Raylib-go with `-tags wayland`).
- From inside the same process, on the **same `wl_display` connection** the
  toolkit uses, bind `zwp_pointer_gestures_v1`, get our own `wl_pointer`
  from the existing `wl_seat`, attach a pinch gesture object.
- Listen for `pinch_begin / pinch_update / pinch_end` events.
- Forward `scale` deltas to the camera; gracefully degrade if the
  compositor doesn't advertise the global (X11, macOS, Windows, older
  compositors).

No upstream patches needed. Pure additive code.

## Phase 0 — verified

`cmd/wl-pinch-probe/` opens its own minimal Wayland window (libwayland-client
via cgo + protocol bindings generated by `wayland-scanner`) and subscribes to
`zwp_pointer_gestures_v1`. Run with:

```sh
go run ./cmd/wl-pinch-probe
```

A small dark-red 320×240 window appears. Pinching over it produces:

```
✓ compositor advertises zwp_pointer_gestures_v1 (version 3)
✓ wl_pointer obtained from wl_seat
✓ pinch gesture object attached
pinch BEGIN  fingers=2  serial=…
pinch UPDATE scale=1.086  rotation= -0.27 deg  dx= -2.85  dy= -5.78
pinch UPDATE scale=1.281  …
pinch END    cancelled=0  serial=…
```

**Hyprland delivers `wp_pointer_gestures_v1` v3 with high fidelity** —
clean scale curve from ~0.87 (pinched in) to ~1.29 (pinched out), rotation
and dx/dy tracked, clean END.

Note also tried `rajveermalviya/go-wayland` (pure-Go bindings). It dies with
`broken pipe` on the second roundtrip after binding `wl_compositor` even at
v1 — incompatibility with Hyprland's recent wayland-protocols. cgo +
libwayland-client (the reference implementation) works.

## Phase 1 plan — proposed, not started

Port godbml from Ebiten to **raylib-go** + integrate the gesture path.

1. Add `gen2brain/raylib-go`, build with `-tags wayland`.
2. Move the Phase 0 cgo gesture code into `internal/`. Get the `wl_display*`
   from raylib's GLFW via `glfwGetWaylandDisplay()` instead of opening our
   own connection.
3. Goroutine dispatches gesture events; main loop applies pinch `scale` to
   the camera around the cursor.
4. Mechanical port of UI: `ebiten.Image` → `rl.RenderTexture2D`, camera
   `GeoM` → `rl.Camera2D` (built-in zoom + offset), text → `rl.DrawTextEx` +
   `rl.LoadFontEx`, input → `rl.IsKeyDown` / `rl.GetMouseWheelMove`, etc.
5. Ctrl+Wheel kept as the universal fallback for X11/macOS/Windows.

Scope: ~1-2 weeks for feature parity. `internal/ui/app.go` is 3049 lines;
mostly mechanical, but text rendering and the four overlay editors (cell
edit, group rename, annotation, palette) need real attention.

## Constraints this changes

`PLAN.md` decision #1 currently reads "**Stack: Ebitengine. Pure Go, no
cgo.**" with WASM as a future target. Phase 1 breaks both:

- raylib-go uses cgo to wrap libraylib.
- The gesture code is C (compiled via cgo).
- WASM target is dead under this path (cgo doesn't run in browsers).

If WASM is still a hard requirement, Phase 1 is wrong and the answer is
**Ctrl+Wheel on Ebiten, ship it.**

## Open decision

The fork in the road, deferred:

- **Ship Ctrl+Wheel on Ebiten.** Hours of work. Keeps PLAN.md decision #1
  intact. Loses real touchpad pinch.
- **Phase 1 (raylib-go + gesture cgo).** 1-2 weeks. Real pinch end-to-end
  on Hyprland. Drops the pure-Go / WASM-future commitment.

Probes shipped:

- `cmd/wl-pinch-probe/` — Wayland gesture probe (cgo + libwayland). **Keep**
  — its code is the basis for Phase 1's gesture module.
- `cmd/pinch-probe/` — pure-Go evdev probe. **Discardable** — rejected path,
  kept only as a reference for the permission-model rationale.
